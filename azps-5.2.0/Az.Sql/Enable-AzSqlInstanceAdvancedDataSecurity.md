# Enable-AzSqlInstanceAdvancedDataSecurity

## SYNOPSIS
관리되는 인스턴스에서 고급 데이터 보안을 사용할 수 있습니다.

## 구문

```
Enable-AzSqlInstanceAdvancedDataSecurity [-DoNotConfigureVulnerabilityAssessment] [-AsJob]
 [-DeploymentName <String>] [-InputObject <AzureSqlManagedInstanceModel>] -InstanceName <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## 설명
**Enable-AzSqlInstanceAdvancedDataSecurity** cmdlet을 사용하면 관리되는 인스턴스에서 고급 데이터 보안을 사용할 수 있습니다. 고급 데이터 보안은 관리되는 인스턴스에 대한 데이터 분류, 취약성 평가 및 Advanced Threat Protection을 포함하는 통합 보안 패키지입니다. (취약성 평가를 저장하기 위해 새 저장소 계정이 자동으로 만들어집니다. 저장소 계정이 이 용도로 이전에 만들어진 경우 대신 사용됩니다.)

## 예제

### 예제 1: 관리되는 인스턴스 고급 데이터 보안 사용
```powershell
PS C:\>  Enable-AzSqlInstanceAdvancedDataSecurity `
            -ResourceGroupName "ResourceGroup01" `
            -InstanceName "ManagedInstance01" 

ResourceGroupName            : ResourceGroup01
ManagedInstanceName          : ManagedInstance01
IsEnabled                    : True
```

### 예제 2: 서버 리소스에서 관리되는 인스턴스 고급 데이터 보안 사용
```powershell
PS C:\>  Get-AzSqlInstance `
           -ResourceGroupName "ResourceGroup01" `
           -Name "ManagedInstance01" `
           | Enable-AzSqlInstanceAdvancedDataSecurity

ResourceGroupName            : ResourceGroup01
ManagedInstanceName          : ManagedInstance01
IsEnabled                    : True
```

### 예제 3

관리되는 인스턴스에서 고급 데이터 보안을 사용할 수 있습니다. (자동Generated)

```powershell
<!-- Aladdin Generated Example -->

```
