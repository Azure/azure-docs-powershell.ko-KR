# Set-AzDataLakeStoreItemAcl

## SYNOPSIS
Data Lake Store에서 파일 또는 폴더의 ACL을 수정합니다.

## 구문

```
Set-AzDataLakeStoreItemAcl [-Account] <String> [-Path] <DataLakeStorePathInstance>
 [-Acl] <DataLakeStoreItemAce[]> [-PassThru] [-Recurse] [-Concurrency <Int32>] [-ShowProgress]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## 설명
**Set-AzDataLakeStoreItemAcl** cmdlet은 Data Lake Store의 파일 또는 폴더의 ACL(액세스 제어 목록)을 수정합니다.

## 예제

### 예제 1: 파일 및 폴더에 대한 ACL 설정
```
PS C:\>$ACL = Get-AzDataLakeStoreItemAclEntry -AccountName "ContosoADL" -Path /
PS C:\> Set-AzDataLakeStoreItemAcl -AccountName "ContosoADL" -Path "/MyFiles/Test.txt" -Acl $ACL
```

첫 번째 명령은 ContosoADL 계정의 루트 디렉터리에 대한 ACL을 구한 다음 $ACL 저장합니다.
두 번째 명령은 파일의 ACL을 Test.txt ACL을 $ACL.

### 예제 2: 폴더에 대한 ACL 재발 설정
```
PS C:\>$ACL = Get-AzDataLakeStoreItemAclEntry -AccountName "ContosoADL" -Path /Folder1
PS C:\> Set-AzDataLakeStoreItemAcl -AccountName "ContosoADL" -Path "/Folder2" -Acl $ACL -Recurse -Concurrency 128
```

첫 번째 명령은 ContosoADL 계정의 Folder1 디렉터리에 대한 ACL을 구한 다음, $ACL 변수에 저장합니다.
두 번째 명령은 ACL을 Folder2로 재구성하여 설정하고 하위 $ACL.
