# New-AzDataMigrationService

## SYNOPSIS
Azure Database Migration Service의 새 인스턴스를 만듭니다.

## 구문

```
New-AzDataMigrationService -ResourceGroupName <String> -Name <String> -Location <String> -Sku <String>
 -VirtualSubnetId <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## 설명
이 New-AzDataMigrationService cmdlet은 Azure Database Migration Service의 새 인스턴스를 만듭니다. 이 cmdlet은 기존 Azure 리소스 그룹의 이름, 만들 Azure Database Migration Service의 새 인스턴스에 대한 고유 이름, 인스턴스가 프로비전되는 지역, DMS Worker SKU의 이름 및 서비스가 상주할 Azure Virtual Subnet의 이름을 사용합니다. 사용자가 Azure 로그인 세션의 기본 구독을 지정하거나 -SubscriptionName "MySubscription" Get-AzSubscription 실행해야 하기 때문에 구독 이름에 대한 매개 변수가 없습니다. | Select-AzSubscription 구독을 선택합니다.

## 예제

### 예제 1
```
PS C:\> New-AzDataMigrationService -ResourceGroupName myResourceGroup -Name TestService -Location "Central US" -Sku Basic_2vCores -VirtualSubnetId $virtualSubNetId
```

위의 예제에서는 미국 중부 지역에 TestService라는 Azure Database Migration Service의 새 인스턴스를 만드는 방법을 보여 주며,
